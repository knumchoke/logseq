- Well-formed XML document
	- เอกสาร XML ที่เป็น well-formed คือเอกสาร XML ที่เป็นไปตามกฎและข้อบังคับของ XML ที่กำหนดไว้ ดังนี้:
		- ต้องมี Prolog: เอกสาร XML ควรมี Prolog ที่ระบุเวอร์ชันของ XML และการเข้ารหัส (encoding) ที่ใช้ เช่น:
		  ```xml
		  <?xml version="1.0" encoding="UTF-8"?>
		  ```
		- มีแท็กหลัก: เอกสาร XML ควรมีแท็กหลักที่ครอบรอบข้อมูลทั้งหมดในเอกสาร แท็กหลักเป็นแท็กเปิดและแท็กปิดที่อยู่ด้านบนสุดของเอกสาร และสมบูรณ์ตามลำดับ ตัวอย่าง:
		  ```xml
		  <root>
		    <!-- ข้อมูล XML อยู่ที่นี่ -->
		  </root>
		  ```
		- แท็กต้องตรงกัน: แท็กเปิดและแท็กปิดต้องตรงกัน และไม่สร้างความสับสน ตัวอย่าง:
		  ```xml
		  <element>ข้อมูล</element>
		  ```
		- ความสัมพันธ์ถูกต้อง: แท็กต้องถูกวางตามความสัมพันธ์ของข้อมูล แท็กเปิดต้องปิดในลำดับถูกต้อง ไม่สร้างความขัดแย้งในโครงสร้างข้อมูล.
		- สิ่งห้ามใช้ในข้อมูล: สัญลักษณ์พิเศษเช่น `&`, `<`, `>`, `"`, และ `'` ต้องถูกแทนที่ด้วย entity references เช่น `&lt;`, `&gt;`, `&amp;`, `&quot;`, และ `&apos;` เมื่อปรากฏในข้อมูล.
		- ความสมบูรณ์ของคุณลักษณะ: หากมีการใช้คุณลักษณะ (attributes) ในแท็ก คุณลักษณะต้องถูกระบุให้ถูกต้องตามรูปแบบที่กำหนด และไม่สร้างความขัดแย้ง.
		- การใช้งาน XML Namespace (ถ้ามี): หากมีการใช้งาน XML Namespace ในเอกสาร XML ควรใช้ Namespace ตามที่กำหนดและไม่สร้างความขัดแย้งกับชื่อแท็ก.
- Creating element in XML document
	- General rules for creating new element
		- ต้องตั้งชื่อตามกฏคือ ขึ้นต้นด้วยตัวอักษรอังกฤษ หรือเครื่องหมายดังนี้ `_` `.` `-`
		- ห้ามขึ้นต้นด้วย `xml` หรือ `XML` ตามข้อกำหนดมาตรฐานของ XML
		- เป็น case sensitive `<TITLE> ... </TITLE>` กับ `<title> ... </title>` เป็นคนละ element กัน
		- ต้องมี root element เพียง 1 อัน element อื่นๆ ต้องอยู่ใน root
		- แท็กต้องตรงกัน: แท็กเปิดและแท็กปิดต้องตรงกัน
		- ตำแหน่ง element ที่ซ่อนใน element ต้องมีการเปิดปิดอย่างถูกต้อง
		- Tag ว่าง สามารถเขียนได้ 2 แบบคือ `<title></title>` หรือ `<name/>`
- Attributes of data within elements
	- character data: บรรจุข้อความตัวอักษรอยู่ภายในอิลิเมนต์
	- character reference / general entity reference:
		- Character reference (อ้างอิงตัวอักษร) เป็นวิธีการใช้สัญลักษณ์พิเศษหรืออักขระทางคีย์บอร์ดที่มีความหมายพิเศษในเอกสาร XML, HTML, หรือ SGML เพื่อแสดงอักขระเหล่านั้นในรูปแบบของข้อมูลแทนที่จะใช้ตรง ๆ ในเอกสาร การใช้ character reference ช่วยให้เอกสารมีความสมบูรณ์และมีความถูกต้องตามกฎและข้อบังคับของ XML หรือ HTML และป้องกันการสร้างความสับสนกับไวยากรณ์ของภาษา.
		  
		  Character reference มักประกอบด้วยอักขระ `&` (แทนตัวแปรเปิดของ character reference) และชื่ออักขระหรือรหัสตัวอักษรที่แทนตัวอักษรที่ต้องการใช้. รูปแบบทั่วไปของ character reference ใน XML และ HTML มีดังนี้:
		  
		  1. ใช้ชื่ออักขระ:
			- ใน HTML: ใช้ชื่ออักขระของสัญลักษณ์พิเศษหรือตัวอักษร เช่น `&lt;` แทน `<`, `&gt;` แทน `>`, `&amp;` แทน `&`, `&quot;` แทน `"`, `&apos;` แทน `'` เป็นต้น.
			- ใน XML: ใช้ชื่ออักขระเหมือนกับใน HTML และสามารถใช้ชื่ออักขระเพิ่มเติมที่ถูกกำหนดใน DTD หรือ XML Schema.
			  
			  2. ใช้รหัสตัวอักษร (Numeric Character Reference):
			- ใช้รหัสตัวอักษรที่แทนตัวอักษรในรูปแบบของรหัส ASCII หรือ Unicode เช่น `&#60;` แทน `<`, `&#62;` แทน `>`, `&#38;` แทน `&`, `&#34;` แทน `"`, `&#39;` แทน `'` เป็นต้น.
			  
			  ตัวอย่างการใช้ character reference ใน HTML:
		- `&lt;`: แทน `<`
		- `&gt;`: แทน `>`
		- `&amp;`: แทน `&`
		- `&quot;`: แทน `"`
		- `&apos;`: แทน `'`
		  
		  ตัวอย่างการใช้ character reference ใน XML:
		- `&#60;`: แทน `<`
		- `&#62;`: แทน `>`
		- `&#38;`: แทน `&`
		- `&#34;`: แทน `"`
		- `&#39;`: แทน `'`
		  
		  การใช้ character reference ช่วยให้คุณสามารถแสดงอักขระทางคีย์บอร์ดหรือสัญลักษณ์พิเศษในเอกสาร XML หรือ HTML โดยไม่สร้างความสับสนในการแปลงและการแสดงผลข้อมูลแก่ผู้ใช้หรือเบราว์เซอร์.
	- nested element หมายถึงการวางแท็กหรืออิลิเมนต์ภายในอิลิเมนต์อื่น ๆ ในเอกสาร XML
	  ```xml
	  <person>
	    <name>John Doe</name>
	    <age>30</age>
	    <address>
	      <street>123 Main St</street>
	      <city>New York</city>
	      <zipcode>10001</zipcode>
	    </address>
	  </person>
	  ```
		- ในตัวอย่างนี้, `<person>` เป็นอิลิเมนต์หลัก ซึ่งมี nested element อยู่ภายใน คือ `<name>`, `<age>`, และ `<address>` ซึ่งล้วนเป็นอิลิเมนต์ที่ซ้อนกันภายใน `<person>` โดย `<address>` มี nested element อีก 3 ตัว คือ `<street>`, `<city>`, และ `<zipcode>` นี่คือการใช้งานของ nested element เพื่อแสดงโครงสร้างข้อมูลที่ซับซ้อนและความสัมพันธ์ของข้อมูลระหว่างอิลิเมนต์ภายในเอกสาร XML
- element's attributes คือข้อมูลเพิ่มเติมที่สามารถระบุให้กับอิลิเมนต์ในเอกสาร XML ในรูปแบบ `name="value"`
  ```xml
  <person name="John Doe" age="30" gender="male">
    <address city="New York" zipcode="10001" />
  </person>
  ```
	- ในตัวอย่างนี้, `<person>` มีคุณลักษณะที่ระบุค่า `name`, `age`, และ `gender` ซึ่งช่วยในการระบุข้อมูลเพิ่มเติมเกี่ยวกับคนนี้. คุณลักษณะที่ปรากฏใน `<address>` คือ `city` และ `zipcode` เช่นกัน.
